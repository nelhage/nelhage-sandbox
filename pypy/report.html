Throughput; higher is better. Bold = fastest engine for that row. Everything is int now, so the rows are directly comparable. C (-O2, no SIMD) leads everywhere; the int loops are fast because int-add is 1-cycle latency and gcc if-converts the sentinel skip to a branchless cmov. Among the JITs, PyPy wins the tight numeric loops and V8 wins object method dispatch. (Building C at -O3 lets it auto-vectorize: plain ~5490, sentinel ~4250 M elem/s.)

sentinel sum throughput vs p(sentinel)

Summing 10M ints while skipping a -1 sentinel (if v == -1: continue), sweeping the fraction that are sentinels. Each point is measured in its own process so each engine's JIT compiles fresh for that fraction. Two effects compete: a branch-misprediction penalty that peaks at p = 0.5 (the skip is maximally unpredictable), and a less-work effect as more skips mean fewer adds — and each engine weights them differently.

  • C is dead flat (~2300) — the cmov runs every iteration regardless of the data, so there's no branch to mispredict and no work saved by skipping. The optimizer erased the whole effect.
  • PyPy traces a symmetric U (~1007 → ~217 → ~993): it emits a real guard, so misprediction dominates and bottoms out at p = 0.5.
  • V8 is a tilted U — a dip at ~0.5 from misprediction, but the high-skip end runs faster than the low-skip end because it does fewer adds.
  • CPython rises monotonically (~44 → ~89): interpreter overhead dwarfs the branch, so the only visible effect is less work per skip.
